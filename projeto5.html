Project Overview

This C++ application determines the optimal route between two points in a directed graph using advanced pathfinding algorithms. It implements Dijkstra's algorithm with priority queues for efficient path calculation, and is designed to be extensible for other algorithms like A*, Bellman-Ford, or Johnson's algorithm. The system supports both command-line interactive use and file-based graph inputs, making it versatile for different use cases.

Key Features:

  • Optimized pathfinding: Dijkstra's algorithm implementation with O(E + V log V) complexity using priority queues
  • Custom graph architecture: Flexible adjacency list representation supporting weighted edges and multiple node types
  • Multi-modal routing: Support for different transportation modes with time constraints (road vs pedestrian)
  • Input flexibility: Terminal-based interactive input or file-based graph specification (CSV format)
  • Robust error handling: Comprehensive input validation and graceful error recovery
  • Test-driven development: 100+ unit tests using GoogleTest covering edge cases and performance benchmarks

Technologies Used:

C++17 C11 STL Containers GoogleTest Graph Theory Algorithm Optimization

Challenges & Solutions:

Performance Optimization

Implemented a min-heap based priority queue to achieve optimal O(E + V log V) time complexity for Dijkstra's algorithm, enabling efficient pathfinding even on graphs with 10,000+ nodes.

Flexible Architecture

Designed a modular graph class that separates graph representation from algorithm implementation, making it easy to add new pathfinding methods or graph types.

Testing & Validation

Developed comprehensive test cases including performance benchmarks, edge cases (negative weights, disconnected graphs), and randomized stress tests to ensure reliability.
