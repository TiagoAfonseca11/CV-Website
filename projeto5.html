Project Overview

This C++ application determines the best route between two points in a directed graph. It supports multiple shortest path algorithms like Dijkstra and can be extended with alternatives like Johnson's or Bellman-Ford. Designed for both command-line and file-based inputs, it emphasizes performance and clarity.

Key Features:

  • Fast pathfinding using Dijkstra’s algorithm with priority queues
  • Custom graph class with adjacency list representation
  • Support for road vs pedestrian time constraints (hybrid mode)
  • Terminal-based interactive input or file-based graph input
  • Robust error handling and input validation
  • Unit tests written with GoogleTest for reliability

Technologies Used:

C++ C STL GoogleTest Graphs & Data Structures

Challenges & Solutions:

Efficiency & Scalability

Used a priority queue-based implementation of Dijkstra to ensure performance even on large graphs with thousands of nodes.

Flexible Input Modes

Designed the system to work both interactively and from files, handling diverse use cases for testing and usage.

Code Quality & Testing

Implemented unit tests using GoogleTest to validate edge cases, performance, and correctness of all critical components.
